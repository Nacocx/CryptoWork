# XdCrypto

一个基于 Python 的 DES 加密与 Modbus TCP 联动演示项目。

本项目展示了如何将自实现的 DES 算法用于 Modbus 寄存器数据的加密、写入、读取和解密，适合用于学习对称加密、报文编码以及 Modbus 通信流程。

## 功能

- 自实现 DES 加密与解密
- 将密文转换为 Modbus 寄存器写入服务器
- 服务器侧拦截写入数据并自动解密
- 客户端侧加密、写入、读取、再解密验证流程
- 支持中文与英文文本演示

## 目录结构

- ModbusServer.py：Modbus TCP 服务器，接收寄存器写入并解密显示明文
- ModbusCilent.py：Modbus TCP 客户端，负责加密后写入服务器
- outputColor.py：终端彩色输出工具
- DES/：DES 算法实现与相关矩阵
	- app.py：DES 单独测试入口
	- DES.py：DES 核心实现
	- DES_Matrixs.py：初始置换、S 盒等矩阵

## 环境要求

- Python 3.8+
- pymodbus 2.5.3

## 安装依赖

建议先创建并激活虚拟环境，然后安装依赖：

```bash
pip install pymodbus
```

如果你的环境中还没有 Python 虚拟环境，也可以直接使用系统 Python，但更推荐使用独立环境。

## 运行方式

### 1. 启动 Modbus 服务器

```bash
python ModbusServer.py
```

默认监听：

- 地址：0.0.0.0
- 端口：5020

### 2. 启动客户端

在另一个终端运行：

```bash
python ModbusCilent.py
```

客户端默认会：

1. 连接 127.0.0.1:5020
2. 使用密钥 Secret8! 加密字符串
3. 将密文写入保持寄存器
4. 读取寄存器并解密验证

## DES 测试

如果你只想测试 DES 算法，可以直接运行：

```bash
python DES/app.py
```

该脚本会对多个密钥和多条消息进行加密 / 解密校验。

## 数据处理说明

当前项目中，密文会先转成十六进制字符串，再按两个字符一组打包成 Modbus 寄存器进行传输。

举例：

- 原始密文 bytes → hex 字符串
- hex 字符串 → 寄存器数组
- 寄存器数组 → hex 字符串
- hex 字符串 → bytes → DES 解密

## 注意事项

- DES 已经不适合现代安全场景，仅用于教学和实验
- 密钥必须正好 8 字节，否则会抛出异常
- 如果要在 Windows 上使用 502 端口，通常需要管理员权限；当前项目默认使用 5020，避免权限问题
- 客户端与服务器中的密钥必须一致，否则无法正确解密

## 示例输出

运行后你将看到类似流程：

- 客户端输出原文、密文、写入结果和解密结果
- 服务器输出写入的寄存器值、密文和明文
